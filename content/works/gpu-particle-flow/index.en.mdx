title: "GPU Particle Flow"
summary: "The state of 65,536 particles held in a GPU texture, simulated through a curl-noise flow field"

## Idea

Each particle is held not on the CPU but in a **GPU texture**. Every texel
of a 256×256 floating-point texture stores one particle's position, life,
and seed, and each frame a simulation shader updates the whole texture at
once — all 65,536 of them simultaneously.

## Core technique

Two textures read and written in turn — a **ping-pong FBO** — carry the
state forward. The flow is the curl of a scalar noise field, giving a
divergence-free vector field where particles neither clump nor scatter but
circulate smoothly.

```glsl
vec2 curl(vec2 p) {
  float e = 0.009;
  float dx = potential(p + vec2(e, 0.0)) - potential(p - vec2(e, 0.0));
  float dy = potential(p + vec2(0.0, e)) - potential(p - vec2(0.0, e));
  return vec2(dy, -dx) / (2.0 * e);
}
```

The render pass reads the particle texture directly with `texelFetch` and
draws all 65,536 points with additive blending. Light gathers where they
overlap.

## Interaction

Around the cursor a tangential force is added, so the swarm gets pulled in
and swirls. Requires WebGL2 and floating-point render targets.
