title: "GPUパーティクルフロー"
summary: "65,536個の粒子の状態をGPUテクスチャに収め、カールノイズの流れ場でシミュレート"
---

## アイデア

粒子の一つひとつをCPUではなく**GPUテクスチャ**に収める。256×256の
浮動小数点テクスチャの各テクセルが粒子一つの位置・寿命・シードを保存し、
毎フレーム、シミュレーションシェーダーがテクスチャ全体を一度に更新する
— 6万5千個を同時に。

## 核心テクニック

二枚のテクスチャを交互に読み書きする**ピンポンFBO**が状態を引き継ぐ。
流れはスカラーノイズの回転(curl)でつくり、発散のない — 粒子が固まったり
散ったりせず、なめらかに巡る — ベクトル場になる。

```glsl
vec2 curl(vec2 p) {
  float e = 0.009;
  float dx = potential(p + vec2(e, 0.0)) - potential(p - vec2(e, 0.0));
  float dy = potential(p + vec2(0.0, e)) - potential(p - vec2(0.0, e));
  return vec2(dy, -dx) / (2.0 * e);
}
```

レンダーパスは `texelFetch` で粒子テクスチャを直接読み、6万5千個の点を
加算合成で描く。重なるほど光が集まる。

## インタラクション

カーソルの周りには接線方向の力が加わり、粒子の群れが吸い込まれて渦巻く。
WebGL2と浮動小数点レンダーターゲットが必要。
