## WASM

브라우저에서 고성능 네이티브 코드 실행 가능하게 해주는 이진 포맷 (가상머신)언어

- 웹에서 JS보다 빠르게 실행 가능

- C, C++, Rust, Go 언어로 작성한 코드를 컴파일 -> 웹에서 실행

- `.wasm` 이진 파일로 컴파일되어 크기 작고, 로딩 빠름

- 보안 굿 -> 왜? -> 샌드박스 환경에서 실행, 메모리 안정성

### 샌드박스 환경?

- 브라우저에서 실행되는 코드가 시스템에 직접 접근하지 못하도록 제한된 환경

- 메모리 접근, 파일 시스템 접근 등 제한

- WASM은 샌드박스 환경에서 실행되므로, 악성 코드가 시스템에 직접 접근할 수 없음

### 장점

- 성능 : 게임, 영상 편집, 이미지 처리 등 브라우저에서도 네이티브급 성능 내야하는 경우

- AI, 머신러닝 워크로드 연계 : 웹에서 WASM을 통해 모델을 실행하는 데 사용성 증가. ONNX, TensorFlow Lite 등 지원

- 멀티플랫폼 실행 : 하나의 코드를 여러 환경에서 돌리기 가능, 서버리스 환경 확장

- JS 보완재 : 전통적인 JS만으로는 어렵고 느린 작업 대체 가능성

### 단점

- 디버깅 어려움

- 러닝 커브

- JS보다 낮은 개발 생산성 -> 개선 가능?

- 최신 브라우저에서만 지원

### 한계

- WASM 파일 크기

- JS와 브릿지를 통해서만 접근 가능해서 DOM 직접 접근 케이스 제한

- Go는 계산 로직 위주, UI는 JS/HTML 구성

- 멀티스레드 안됨, OS 호출도 제한적(WASI 환경 시 일부 가능)

### 활용도

- 브라우저에서 Go 로직 실행 가능

- 서버 구축도 WASM을 서버리스 런타임으로 사용하면 가능(WASI)

- 앱 개발은 불가능

- WASM 기반 AI 오프라인 번역기 ..? -> HuggingFace 에서 모델 받아서 활용

## Golang으로 WASM 간단 구축

Golang이 공식적으로 WASM 지원 발표

```bash
GOOS=js GOARCH=wasm go build -o main.wasm
```
