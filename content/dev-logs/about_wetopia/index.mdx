## 프로젝트 진행상황

### <u>Frontend</u>

#### ✔️ 배포

- AWS Amplify 배포 예정

#### ✔️ 기능

주가 조회, 적정가 자동 계산, 미국 주식 관련주 기사 실시간 열람 기능을 생각하며 프로젝트를 시작하였으나, '이걸 누가 쓰지' 싶은 의구심을 시작으로 그냥 내가 관심있는 투자를 위한 프로젝트인 점 말고는 메리트가 없어보임. + 이미 더 편한 툴이 있는만큼 이걸 왜 쓰는데 라는 생각도.

DB에 Cron으로 일간 주가 보관한거를 가져와서 차트로 조회하는 기능만 구현된 상태.

관심 있는 분야 + 유용한 기능 + 편의성 에 대한 고민을 더 해보는 중.

#### 마인드맵핑

- '느린 SNS'를 주제로 '하루에 한 번만 -을 할 수 있는 서비스'

- 국내 부동산 시장은 자동 분석, 리포트 관련 서비스

#### ✔️ 기술 스택

Framework: Next.js 15.2 (App Router)

State Management: Recoil, React Query

Type Checking: TypeScript

Package Manager: Yarn Berry (PnP, loose mode)


### <u>Backend</u>

#### ✔️ 배포

App Runner 배포 완료

- **런타임**: Node.js 20
- **소스 디렉토리**: `/`
- **빌드 명령어**:
  ```bash
  yarn install && yarn build
  ```
- **시작 명령어**:
  ```bash
  node dist/main.js
  ```
- **포트**: 3000 -> 4000 변경
- **환경 변수 등록 필수**

#### ✔️ 기술 스택

Framework : Nest.js

Database : PostgreSQL (Neon.tech에 배포해둔 상태)

ORM: TypeORM

Scheduler: `@nestjs/schedule`

HTTP Client: `@nestjs/axios`

- GraphQL로 바꿀 예정

Hosting: AWS App Runner

#### ✔️ 디렉토리 구조

```bash
wetopia-backend
├── src
│   ├── app.module.ts         # 루트 모듈
│   ├── main.ts               # 앱 진입점
│   ├── config                # 환경변수 설정 모듈
│   │   └── config.module.ts
│   ├── stocks                # 주가 관련 도메인
│   │   ├── dto               # 요청 및 응답 DTO
│   │   ├── entities          # TypeORM 엔티티
│   │   ├── stocks.module.ts
│   │   ├── stocks.service.ts
│   │   ├── stocks.controller.ts
│   ├── external              # 외부 API 연동 (TwelveData)
│   │   └── twelve-data.service.ts
│   ├── common                # 공용 유틸, 필터, 예외 등
│   │   ├── filters
│   │   ├── interceptors
│   │   ├── utils
├── test                     # 테스트 코드
├── .env                     # 환경 변수 파일
├── tsconfig.json
├── package.json
```
