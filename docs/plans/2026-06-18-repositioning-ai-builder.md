## 0. 한 줄 요약

지금 사이트는 **구직용 프론트엔드 포트폴리오**다. 너는 5개월간 30+개의 AI 시스템을
혼자 만들어 굴린 **빌더**다. 둘은 정체성이 다르다. 그리고 **"GitHub으로 강화"는
그대로 하면 역효과** — 공개 repo가 옛날 프론트엔드 코드로 도배돼 있어서, 자동으로
끌어오면 정확히 벗어나고 싶은 이미지가 강화된다. 해법은 *자동 피드가 아니라 큐레이션*.

> 아니라 *한 도메인의 곁가지*로 내려야 한다.

## 2. 핵심 제약 — GitHub 현실 점검 (가장 중요)

`gh`로 실제 확인한 결과:

**공개(public) non-fork repo 24개 — 거의 전부 옛날 FE / 일반:**

`nextjs-base`, `dev-quickstart`, `Connecty-*`(2021), `DayByDay`, `API_Server` …

**비공개(private) non-fork repo 37개 — 진짜 AI 작업 전부:**
`plato`, `charlie`, `nexus`, `herald`, `oddless`, `reaper`, `cherry`, `cura`, `morph`,
`tessera`, `quant`, `lotto-bot`, `coin-bot`, `stock-bot`, `savvy`, `agora`, `autopilot` …

### 결론: "GitHub API로 repo 자동 표출" 전략은 폐기

자동으로 끌어오면 보이는 건 **공개 repo = 옛날 프론트엔드**다. 벗어나려는 이미지를
오히려 강화한다. (리서치: *"site is the container, the work is the content"* — 컨테이너를
자동화해봐야 내용물이 옛날 것이면 의미 없음.)

### "깁으로 강화"의 재정의 — 2층 증명 구조

| 층 | 무엇 | 대상 |
|---|---|---|
| **공개 GitHub 층** | 안전하게 오픈소스 가능한 *소수*를 골라 public 전환 → repo 링크로 증명 | `oddless`, `plato`, (정제된) `quant` 등 비밀/키 없는 도구 |
| **블로그 케이스스터디 층** | 코드 공개 불가한 다수 → 빌드로그/케이스스터디로 증명 (코드 없이 결과·아키텍처·메트릭) | 트레이딩(`charlie`/`kairos`/`reaper`), 상용 파이프라인 |

핵심: **GitHub은 "공개할 소수"의 증명 레이어, 블로그는 "나머지 다수"의 증명 레이어.**
자동 피드 X, 큐레이션 O.

#### 공개 후보 우선순위 (안전성 × 매력도)

1. **`oddless`** — Suno→YouTube 렌더러. 비밀/키 없음, 깔끔한 CLI 도구. **즉시 공개 가능.**
2. **`plato`** — 4 페르소나(Maya/Theo/Iris/Sam) 아이디어 토론 엔진. 서사가 매력적, 비밀 없음. **공개 시 화제성 높음.**
3. **`quant`** — bias-guarded 리서치 엔진 + **정직한 부정 결과**(FINDINGS.md: "none is a confirmed edge"). 알파/키 제거하면 *방법론 자체*가 공개 자산. → **메인 시그니처**.
4. **`nexus`** — 음성 비서 + 코딩 오케스트레이터. `docs/SECURITY.md` 있으니 키·로컬경로 점검 후.
5. 트레이딩 실전봇(`charlie`/`reaper`/`coin-bot`/`stock-bot`) — **공개 금지**, 케이스스터디로만.

## 3. 새 포지셔닝

### 정체성 (직함 아님, 동사)
> **"혼자서 AI 시스템을 만들고 굴린다 — 트레이딩, 에이전트, 생성형 미디어, 게임까지."**

세 가지 축으로 신뢰를 만든다 (리서치의 신뢰 3요소):

1. **카덴스/규모 숫자** — "5개월(2026.02~06), 30+ 시스템." 진짜 숫자가 가장 강한 증명 레버. (levels.io "12 startups in 12 months" 패턴)
2. **깊이 앵커 1개** — 폭이 흩어짐으로 안 읽히려면 한 도메인 깊이가 필요. → **퀀트/트레이딩**. `quant`의 편향 가드 방법론 + 정직한 부정 결과는 *진짜 진지한 작업*의 증거.
3. **정직성 시그널** — `quant`의 "엣지 없음" 결론, `charlie`의 봉인(완주·재현 실패→종료). 리서치: *"named failures가 가장 위조 불가능한 신뢰 신호"*. 실패를 명시하는 빌더가 이긴다.

### 폭(breadth)을 약점이 아니라 강점으로 — "cross-domain arbitrage"
> "트레이딩 시스템의 피드백 루프를 에이전트에, 게임의 상태관리를 음성 UX에 옮긴다.
> 도메인이 30개여도 패턴은 하나 — 자율 루프 · 에이전트 · 파이프라인."

폭을 *나열*하면 산만, *통합*으로 말하면 메타스킬. 독자에게 관통하는 한 문장을 직접 쥐여줄 것.

## 4. 새 IA + 랜딩 설계

### 네비게이션
```
0biglife.   |   Lab   ·   Writing   ·   About
```
- **Lab** (또는 Projects) — 새 홈. 프로젝트 쇼케이스.
- **Writing** (현 `/log`) — 빌드로그/케이스스터디/딥다이브.
- **About** — `/introduction`을 *빌더 소개*로 재작성. 자기소개서 PDF는 삭제하거나 조용한 `/resume` 링크 한 줄로.
- WebGL 갤러리 = 죽이지 말고 **Lab 안의 한 필터(`#interactive`)** 또는 `/lab/playground`로 강등.

### 홈(`/`) = 빌더 랜딩, 위→아래 순서
1. **히어로** — 동사-우선 정체성 + "now building" + 숫자 (아래 카피 옵션)
2. **Featured Projects** — 상위 6개, **status 뱃지**(live / shipped / sealed / experiment) + **도메인 필터**(trading · agents · media · voice · games · web3). 기존 `WorkCard`/`works.ts` 데이터모델 재사용 확장.
3. **Featured Writing** — 시그니처 글 3개 (예: "5개월간 AI 시스템 30개 — 무엇이 살아남았나", "내 퀀트 엔진은 알파가 없다는 걸 증명했다")
4. **(접힌) 전체 아카이브** — status·domain 필터되는 levels.io/projects 스타일 리스트. 죽은 프로젝트도 보여주는 정직함이 곧 브랜드.

> 그리드 하나로 30개 늘어놓지 말 것 → 규모가 오히려 싸 보임. **상위 큐레이션 + 깊은 곳에 필터되는 전체 리스트.**

### 히어로 카피 옵션 (ko 우선, 셋 중 택1 후 다듬기)

**A — 숫자 증명형 (levels.io 스타일, 가장 셈)**
> 혼자서 AI 시스템을 만들고 굴립니다.
> 2026년 2월부터 트레이딩·에이전트·생성형 미디어·게임까지 30개+.
> 대부분은 실험으로 끝났고, 몇 개는 지금도 돌아갑니다.

**B — 메타스킬형 (폭을 강점으로)**
> 도메인이 뭐든 AI 제품을 *출시하는 법*을 압니다.
> 트레이딩 엔진, 음성 비서, 콘텐츠 파이프라인 — 패턴은 하나: 자율 루프.

**C — 정직한 빌더형 (신뢰 시그널)**
> AI 시스템을 만들고, 되는지 정직하게 측정합니다.
> 내 퀀트 엔진이 찾은 결론은 "엣지 없음"이었습니다. 그게 이 사이트입니다.

→ 추천: **A를 메인 히어로 + C의 정직함을 바로 아래 한 줄**로. ("open to work" 류 구직 언어는 전부 제거.)

## 5. 글쓰기 전략 (Writing/Log)

리서치 결론: **결과물을 출시하고 + 그 영수증을 쓴다.** 권위는 빌드로그 + "X를 만들었다"
워크스루 + evals/메트릭 + 포스트모템에서 나온다. FE 튜토리얼 글이 아니라.

**즉시 쓸 시그니처 글 (있는 자산 기반, 공개 안전):**

1. **"내 퀀트 엔진은 알파가 없다는 걸 증명했다"** — `quant` FINDINGS.md 기반.
   생존편향 +116pp 환상, 단일 2021 레짐 아티팩트, 자유 데이터 ~2016 바닥.
   *결론이 부정인 글*이라 비밀 노출 0, 신뢰 100. → **첫 번째로 써라.**
2. **"5개월간 AI 시스템 30개 — 무엇이 살아남았나"** — 전체 빌드 회고. status별 분류(live/sealed/experiment), 폭을 cross-domain arbitrage로 프레이밍.
3. **"4명의 AI 직원에게 내 아이디어를 토론시킨다"** — `plato` 빌드로그. 페르소나 설계.
4. **"음성으로 Claude Code 워커를 띄운다"** — `nexus` 코딩 오케스트레이터 딥다이브.

**비공개 작업 보여주는 법 (코드 없이):**
Problem→Approach→Result 케이스스터디 · 아키텍처 다이어그램(가장 NDA-safe) ·
*묘하게 구체적인* 메트릭(반올림 X) · 데모 영상/GIF · "무엇이 깨졌나" 회고.
트레이딩은 equity curve 모양 · Sharpe · max drawdown 정직하게, **시그널/엔트리 로직은 절대 X**.
